# 用户数据表设计文档

## 一、表结构概览

| 表名 | 用途 |
|------|------|
| users | 用户基础信息与权限 |

---

## 二、users 表设计

### 2.1 表结构

| 字段名 | 类型 | 约束 | 默认值 | 说明 |
|--------|------|------|--------|------|
| id | VARCHAR(36) | PRIMARY KEY | UUID | 用户唯一标识 |
| username | VARCHAR(50) | NOT NULL, UNIQUE | - | 用户名（登录账号） |
| password_hash | VARCHAR(255) | NOT NULL | - | 密码哈希值 |
| display_name | VARCHAR(50) | NOT NULL | - | 显示名称 |
| role | ENUM | NOT NULL | 'calculator' | 角色：calculator/auditor/admin |
| enabled | TINYINT(1) | NOT NULL | 1 | 是否启用：1启用/0禁用 |
| created_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 更新时间 |
| last_login_at | DATETIME | NULL | NULL | 最后登录时间 |

### 2.2 角色枚举值

| 角色值 | 显示名称 | 权限范围 |
|--------|----------|----------|
| calculator | 计算员 | 查看桥梁、执行计算、查看历史 |
| admin | 管理员 | 全部功能 + 审核、报告、系统设置、用户管理 |

### 2.3 索引设计

| 索引名 | 字段 | 类型 | 用途 |
|--------|------|------|------|
| pk_users | id | 主键 | 唯一标识 |
| uk_username | username | 唯一索引 | 登录查询 |
| idx_role | role | 普通索引 | 按角色筛选 |
| idx_enabled | enabled | 普通索引 | 按状态筛选 |

---

## 三、SQL 建表语句

```sql
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY COMMENT '用户唯一标识',
    username VARCHAR(50) NOT NULL COMMENT '登录账号',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希值',
    display_name VARCHAR(50) NOT NULL COMMENT '显示名称',
    role ENUM('calculator', 'admin') NOT NULL DEFAULT 'calculator' COMMENT '用户角色',
    enabled TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    last_login_at DATETIME NULL COMMENT '最后登录时间',
    
    UNIQUE KEY uk_username (username),
    INDEX idx_role (role),
    INDEX idx_enabled (enabled)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 五、业务操作对应SQL

### 5.1 新增用户

```sql
INSERT INTO users (id, username, password_hash, display_name, role, enabled)
VALUES (UUID(), ?, ?, ?, ?, 1);
```

### 5.2 查询用户列表

```sql
SELECT id, username, display_name, role, enabled, created_at, last_login_at
FROM users
ORDER BY created_at DESC;
```

### 5.3 修改角色

```sql
UPDATE users SET role = ?, updated_at = NOW() WHERE id = ?;
```

### 5.4 重置密码

```sql
UPDATE users SET password_hash = ?, updated_at = NOW() WHERE id = ?;
```

### 5.5 启用/禁用用户

```sql
UPDATE users SET enabled = ?, updated_at = NOW() WHERE id = ?;
```

### 5.6 删除用户

```sql
DELETE FROM users WHERE id = ?;
```

### 5.7 用户登录验证

```sql
SELECT id, username, password_hash, display_name, role, enabled
FROM users
WHERE username = ? AND enabled = 1;
```

### 5.8 更新最后登录时间

```sql
UPDATE users SET last_login_at = NOW() WHERE id = ?;
```

## 七、安全考虑

| 安全项 | 实现方式 |
|--------|----------|
| 密码存储 | 使用 bcrypt 哈希，不存储明文 |
| 登录限制 | 连续失败5次锁定账号30分钟（可扩展） |
| 会话管理 | JWT Token，设置合理过期时间 |
| 操作日志 | 记录敏感操作（可选扩展日志表） |
