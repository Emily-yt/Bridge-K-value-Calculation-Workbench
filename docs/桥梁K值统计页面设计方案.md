# 桥梁K值统计页面设计方案

## 1. 设计目标

从桥梁检修工程师的实际工作场景出发，以**单座桥梁**为核心分析单元，清晰展示**桥梁→孔跨→K值**的层级关系，帮助工程师快速定位问题桥梁、识别短板孔跨、追踪承载能力变化趋势。

---

## 2. 数据模型关系

### 2.1 核心实体关系

```
桥梁 (Bridge)
  ├── id, bridgeName, lineName, bridgeNo, centerMileage
  ├── spanType, spanCount, buildYear, operationStatus
  └── spans: BeamSpan[] (孔跨数组)
        ├── index: 孔跨序号(1-based)
        ├── beamLength, beamHeight, beamCenterDist
        └── beamType: 梁型图号

K值计算记录 (KValueCalculation)
  ├── bridgeId: 关联桥梁
  ├── spanIndex: 计算的孔跨序号
  ├── beamType, spanLength
  ├── input: 计算输入参数
  ├── output: { k1, k2, k3, k4, kFinal }
  └── createTime: 计算时间
```

### 2.2 桥梁K值计算逻辑

```
桥梁K值 = min(所有已计算孔跨的 kFinal)

判定规则:
  - 0个孔跨已计算 → 桥梁K值 = "未评估"
  - N个孔跨已计算 → 桥梁K值 = min(第1孔kFinal, ..., 第N孔kFinal)

短板孔跨 = kFinal 等于桥梁K值的孔跨（可能多个）
控制分项 = 决定 kFinal 的那个 Ki（kFinal = min(K1, K2, K3, K4)）
```

### 4.3 组件详细说明

#### 4.3.1 时间范围筛选

| 选项 | 说明 |
|------|------|
| 全部时间 | 使用所有历史计算记录 |
| 近30天 | 只统计最近30天内的计算记录 |
| 近90天 | 只统计最近90天内的计算记录 |
| 近一年 | 只统计最近365天内的计算记录 |

**影响范围**：所有统计卡片、排名表、图表均按筛选后的计算记录重新计算。

#### 4.3.2 统计卡片

| 卡片 | 计算逻辑 | 点击行为 |
|------|---------|---------|
| 桥梁总数 | bridges.length | 无 |
| 问题桥梁 | 桥梁K值 < 1.5 的桥梁数量 | 跳转到明细Tab，自动筛选K<1.5 |
| 计算覆盖 | 至少1个孔跨已计算的桥梁数 / 总桥梁数 | 无 |
| 孔跨覆盖 | 已计算孔跨数 / 所有桥梁总孔跨数 | 无 |

#### 4.3.3 桥梁K值排名表

**排序规则**：按桥梁K值升序（问题桥梁排在前面），未评估桥梁排在最后。

**列定义**：

| 列名 | 说明 |
|------|------|
| 排名 | 1, 2, 3... |
| 桥梁名称 | bridgeName，过长时截断显示... |
| 线路 | lineName |
| 桥梁K值 | 该桥所有已计算孔跨的kFinal最小值，保留4位小数 |
| 等级 | 根据桥梁K值显示颜色标签 |
| 已算/总孔 | 该桥已计算孔跨数 / 总孔跨数 |

**行点击**：跳转到「桥梁分析」Tab，自动选中该桥梁。

#### 4.3.4 K值等级分布饼图

**统计维度**：按**桥梁数量**统计（不是按计算次数）。

**数据计算**：
```javascript
{
  safe:   bridges.filter(b => b.bridgeKValue >= 2.0).length,
  normal: bridges.filter(b => b.bridgeKValue >= 1.5 && b.bridgeKValue < 2.0).length,
  low:    bridges.filter(b => b.bridgeKValue >= 1.0 && b.bridgeKValue < 1.5).length,
  danger: bridges.filter(b => b.bridgeKValue < 1.0).length
}
```

**图例**：显示等级名称、桥梁数、百分比。

#### 4.3.5 桥梁K值趋势折线图

**图表类型**：横向或纵向折线图（根据桥梁数量自适应）。

**数据准备**：
1. 取所有桥梁，按桥梁K值升序排列
2. X轴：桥梁名称（或序号）
3. Y轴：桥梁K值
4. 每个数据点：{ name: 桥梁名称, value: 桥梁K值, level: 等级 }

**视觉编码**：
- 数据点颜色：按K值等级（🟢🔵🟡🔴）
- 参考线：K=1.0（红色虚线）、K=1.5（黄色虚线）
- 悬停提示：显示桥梁名称、K值、等级、已算孔跨数

**工程师价值**：
- 一眼看出所有桥梁的K值分布形态
- 快速识别"有多少桥在危险线以下"
- 发现K值聚集区间（如大部分桥集中在1.5-2.0）

### 5.3 组件详细说明

#### 5.3.1 桥梁选择器

- 下拉选择框，显示桥梁名称和编号
- 支持搜索（输入桥梁名称/编号快速定位）
- 切换桥梁时，下方所有组件联动刷新

#### 5.3.2 桥梁信息卡片

**基础信息行**：
- 线路、桥梁编号、中心里程
- 孔跨组成（spanType）、建成年度、运营状态

**K值状态区**（重点突出）：

| 字段 | 说明 |
|------|------|
| 桥梁K值 | 大号字体显示，保留4位小数 |
| 等级标签 | 带颜色的等级名称（如"不满足运营 🔴"）|
| 短板孔跨 | 显示kFinal等于桥梁K值的孔跨序号（如"第4孔"）|
| 已计算 | "3/16 孔"表示已计算3孔，共16孔 |
| 最近计算 | 该桥最近一次计算的日期 |
| 控制分项 | 决定短板孔跨kFinal的分项名称（如"K1 正截面抗弯强度"）|

#### 5.3.3 孔跨K值一览

**展示形式**：横向条形图或热力条，每个孔跨一行。

**每行结构**：
```
[孔跨序号] [进度条/热力条] [K值或状态]
```

**颜色编码**：

| 状态 | 颜色 | 说明 |
|------|------|------|
| 未计算 | ⬜ 灰色 | 该孔跨暂无计算记录 |
| K≥2.0 | 🟢 绿色 | 安全充裕 |
| 1.5≤K<2.0 | 🔵 蓝色 | 满足运营 |
| 1.0≤K<1.5 | 🟡 黄色 | 承载偏低 |
| K<1.0 | 🔴 红色 | 不满足运营 |

**交互**：
- 点击某孔跨行 → 展开/收起该孔的K1~K4分项详情
- 短板孔跨（决定桥梁K值的孔）用箭头或边框高亮标注

**工程师价值**：
- 一眼看出"哪些孔跨还没算"
- 快速识别"哪个孔跨是短板"
- 发现"是否多个孔跨都有问题"

#### 5.3.4 K1~K4 分项详情表

**展示对象**：该桥梁所有已计算孔跨。

**列定义**：

| 列名 | 说明 |
|------|------|
| 孔跨 | 第X孔 |
| 梁型 | beamType |
| K1抗弯 | 正截面抗弯强度K值 |
| K2抗裂 | 正截面抗裂性K值 |
| K3应力 | 正截面应力K值 |
| K4抗剪 | 斜截面抗剪K值 |
| K最终 | min(K1,K2,K3,K4)，决定孔跨等级 |
| 控制 | 决定K最终的那个分项（如K1），高亮显示 |

**视觉处理**：
- 每个K值单元格根据数值显示背景色（绿/蓝/黄/红）
- 「控制」列用粗体或标签高亮

#### 5.3.5 历史趋势图

**数据准备**：

## 8. 关键计算逻辑

### 8.1 桥梁K值计算

```typescript
function calculateBridgeKValue(
  bridge: Bridge,
  calculations: KValueCalculation[]
): BridgeKValueResult {
  // 筛选该桥的所有计算记录
  const bridgeCalcs = calculations.filter(c => c.bridgeId === bridge.id);
  
  // 按孔跨分组，取每孔最新的计算记录
  const spanLatestCalcs = new Map<number, KValueCalculation>();
  for (const calc of bridgeCalcs) {
    const existing = spanLatestCalcs.get(calc.spanIndex);
    if (!existing || new Date(calc.createTime) > new Date(existing.createTime)) {
      spanLatestCalcs.set(calc.spanIndex, calc);
    }
  }
  
  // 提取各孔K值
  const spanKValues: SpanKValue[] = [];
  for (const [spanIndex, calc] of spanLatestCalcs) {
    spanKValues.push({
      spanIndex,
      kFinal: calc.output.kFinal,
      k1: calc.output.k1,
      k2: calc.output.k2,
      k3: calc.output.k3,
      k4: calc.output.k4,
      controlComponent: getControlComponent(calc.output) // 'K1'|'K2'|'K3'|'K4'
    });
  }
  
  // 计算桥梁K值
  if (spanKValues.length === 0) {
    return {
      bridgeKValue: null,
      level: 'unevaluated',
      spanKValues: [],
      weakSpanIndices: [],
      controlComponent: null
    };
  }
  
  const kValues = spanKValues.map(s => s.kFinal);
  const bridgeKValue = Math.min(...kValues);
  const level = getKValueLevel(bridgeKValue);
  
  // 找出短板孔跨
  const weakSpanIndices = spanKValues
    .filter(s => s.kFinal === bridgeKValue)
    .map(s => s.spanIndex);
  
  // 短板孔跨的控制分项
  const weakSpan = spanKValues.find(s => s.spanIndex === weakSpanIndices[0]);
  
  return {
    bridgeKValue,
    level,
    spanKValues,
    weakSpanIndices,
    controlComponent: weakSpan?.controlComponent || null
  };
}

function getControlComponent(output: KValueOutput): 'K1'|'K2'|'K3'|'K4' {
  const minK = Math.min(output.k1, output.k2, output.k3, output.k4);
  if (minK === output.k1) return 'K1';
  if (minK === output.k2) return 'K2';
  if (minK === output.k3) return 'K3';
  return 'K4';
}
```

### 8.2 时间筛选逻辑

```typescript
function filterByTimeRange(
  calculations: KValueCalculation[],
  timeRange: 'all' | 'month' | 'quarter' | 'year'
): KValueCalculation[] {
  if (timeRange === 'all') return calculations;
  
  const now = new Date();
  const days = { month: 30, quarter: 90, year: 365 }[timeRange];
  const cutoff = new Date(now.getTime() - days * 24 * 60 * 60 * 1000);
  
  return calculations.filter(c => new Date(c.createTime) >= cutoff);
}
```

## 9. 与现有实现的差异总结

| 维度 | 现有实现 | 新方案 |
|------|---------|--------|
| **分析粒度** | 全局聚合，不区分单桥 | **以桥梁为核心**，支持单桥深入分析 |
| **孔跨关系** | 完全忽略孔跨 | **孔跨K值一览** + 短板标注 + 控制分项 |
| **趋势分析** | 全局月度计算次数 + 平均K值 | **按桥梁**展示K值随计算推进的变化 |
| **Tab结构** | 概览、K值分析、趋势、明细 (4个) | 总览、桥梁分析、明细 (3个) |
| **全局筛选** | 有（时间范围） | **只在总览Tab保留时间筛选** |
| **总览图表** | K值分布柱状图 | **K值排名表** + K值分布饼图 + **桥梁K值趋势折线图** |
| **线路分布** | 有 | **去掉** |
| **趋势对比** | 无 | **去掉**（按用户要求） |
| **工程师关注点** | 数据展示 | **问题定位**：哪座桥、哪个孔、哪个分项 |
