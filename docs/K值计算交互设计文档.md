## 一、交互流程总览

```
点击"开始计算" → 参数验证 → 计算Loading → 抽屉内简化结果 → 详情弹窗（可选）→ 保存/报告
```

**设计原则：**
- 抽屉保持简洁，快速反馈核心结果
- 详情弹窗空间充足，展示完整信息
- 操作路径清晰，减少用户决策成本

---

## 二、参数验证阶段

### 2.1 验证规则

| 场景 | 交互表现 |
|------|----------|
| 参数完整 | 按钮可点击，进入计算 |
| 必填项缺失 | 输入框红色边框 + 错误提示，按钮禁用 |
| 曲线梁未填半径 | 提示"曲线梁需填写曲线半径" |

### 2.2 验证时机

- 实时验证：用户输入时即时反馈
- 提交验证：点击计算按钮时再次校验

## 六、状态与颜色规范

### 6.1 K值状态判断

| K值范围 | 状态 | 抽屉简化显示 | 详情弹窗提示 | 颜色 |
|---------|------|--------------|--------------|------|
| K ≥ 2.0 | 优秀 | ✓ K=2.15 | 承载能力充裕 | 绿色 |
| 1.5 ≤ K < 2.0 | 合格 | ✓ K=1.68 | 满足运营要求 | 绿色 |
| 1.0 ≤ K < 1.5 | 预警 | ⚠️ K=1.32 | 承载能力偏低，建议加强监测 | 橙色 |
| K < 1.0 | 不合格 | ❌ K=0.95 | 不满足运营要求，需立即处理 | 红色 |

### 6.2 状态标识规范

| 状态 | 标识 | 颜色代码 |
|------|------|----------|
| 优秀/合格 | ✓ | #52C41A (绿色) |
| 预警 | ⚠️ | #FA8C16 (橙色) |
| 不合格 | ❌ | #F5222D (红色) |

### 7.4 结论文案模板

根据K值范围，系统自动生成结论：

| K值范围 | 结论文案 |
|---------|----------|
| K ≥ 2.0 | 该桥梁检定承载系数K=X.XX，满足规范要求，承载能力充裕。 |
| 1.5 ≤ K < 2.0 | 该桥梁检定承载系数K=X.XX，虽不满足规范≥2.0的要求，但K≥1.5，满足运营要求。建议加强日常监测。 |
| 1.0 ≤ K < 1.5 | 该桥梁检定承载系数K=X.XX，承载能力偏低。建议限制通行荷载，加强监测，尽快安排加固处理。 |
| K < 1.0 | 该桥梁检定承载系数K=X.XX，不满足运营要求。建议立即限制通行，安排专项检测和加固处理。 |

## 八、保存与报告流程

    ↓

└─────────────────────────────┘
    ↓

### 8.3 操作按钮说明

| 按钮 | 功能 | 后续动作 |
|------|------|----------|

| 保存结果 | 保存计算记录 | 显示成功提示，关闭抽屉 |
| 保存并生成报告 | 保存并跳转 | 关闭抽屉，跳转报告预览页 |

## 九、异常场景处理

| 场景 | 交互处理 |
|------|----------|
| 计算超时 | 弹窗提示"计算超时，请重试"，保留已输入参数 |
| 参数异常 | 标红异常参数，提示"XX参数超出合理范围" |
| 网络错误 | 提示"网络异常，请检查连接后重试" |
| 历史记录冲突 | 提示"已存在相同参数的计算记录，是否覆盖？" |

## 十一、设计优势总结

| 优势 | 说明 |
|------|------|
| 抽屉保持简洁 | 不需要展示过多信息，用户快速看到K值和结论即可 |
| 详情弹窗空间充足 | 可以完整展示K1~K4、参数、历史对比等 |
| 操作路径清晰 | 满意→保存；想看详情→弹窗；要报告→下载 |
| 异常处理完善 | 覆盖超时、网络错误、参数异常等场景 |
